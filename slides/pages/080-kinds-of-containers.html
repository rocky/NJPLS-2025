Kinds of Containers

  • a file in a file system,
  • a stream from a socket or piped into a compiler,
  • a piece of an "archive"-format file: Java classfile jar, Python wheel (zip), or machine-code library, etc.,
  • a string or AST data-structure created inside a program at runtime, e.g. HTML templating system for Web framework, an AI auto-learning system

Program source code comes from somewhere. It is useful for the programmer and IDE's to know where it came from. So what are some possibilities?

As I said before, a file in the file system is the most common possibility.

A problem I see in many systems, is a weakness in being able to deal with positions when working containers other than a file in a filesystems.

So let's go into other kinds of containers.
