Kinds of Containers

Where can a program come from?

  • a file in a file system,
  • a stream from a socket or piped into a compiler,
  • a piece of an "archive"-format file: Java classfile jar, Python wheel (zip), or machine-code library, etc.,
  • a string or AST data-structure created inside a program at runtime, e.g. HTML templating system for Web framework, an AI auto-learning system

Program source code comes from somewhere. It is useful for the programmer and IDE's to know where it came from. So what are some possibilities?

As I said before, a file in the file system is the most common possibility.

A problem I see in many systems, is a weakness in being able to deal with positions when working containers other than a file in a filesystem.

So let's go into other kinds of containers.

[DOWN fragment to stream] 8.1 ⟶ 8.1

Interactive input; or more generally a stream from a socket which might happen in remote execution. Or a stream piped into a compiler. The weakness I see here is an inability of the system to be able to show the source text in an IDE. Here, input is often one line long. So it is useful to be able to give position information in finer granularity just a line.

a piece of an archive-format file. Java "class file" or jar files are like this. In Python, wheels are zip files.

Another possibility is a string or AST data-structure created inside a program at runtime; for example an HTML templating system for Web framework, or self-modifying code in an auto-learning system.

Again, IDE's and debuggers are sometimes weak in being able to show source text when the container is of this type.
