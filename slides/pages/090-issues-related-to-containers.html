Out-of-Sync Issues that can arise with Containers

  • Compiled source text is not that same as text inside file that IDE uses
    • File is ambiguous, directory or version-control-repository is different.
    • Version of file is different
  • Having access to source text

As I said, the reason we want to store a container name is so that debuggers can find the source text or so report that container in an runtime error message. Often the file mentioned will be opened and the source position will be shown in a debugger or in such an error message.

But this opens up the possibility for another problem: the source text that the compiler or interpreter read the program isn't the same as what the programmer or rather programmer's IDE sees.

Similarly, the container or version control repository are the same but the version of the file is different. This often happens when the file is source text gets modified after the program starts running.

Another common situation is that the file is in version control but the version that was compiled and the version that is currently checked out is a different.

Out-of-Sync Issues Remedies for Containers

  • SIP hash of source text
  • Storing source text inside debug information
  • Passing source text over the network (for remote execution)

One way to detect this problem is to store a checksum or hash of the source text. Python has the ability to store a SIP hash inside bytecode for the purpose of making sure source code matches. Python selected the SIP hash to be relatively fast, while also keeping the possibility of for hash collisions low.

Another possibility that is sometimes used, is to store the source text inside a debug file. The debug tables for Etherium Solidity work this way. The downside of this is that it might lead to huge debug files. Solidity programs however are generally very short

Another technique used a bit in remote debugging is to have the the source text sent across the network on request. So a local IDE doesn't try to access the source text n certain cases, even if the source text exists locally. A downside of this is that a lot of information might be passed over the network if you are not careful and information isn't cached.
